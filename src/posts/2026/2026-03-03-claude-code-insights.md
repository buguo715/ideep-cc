大家好，今天介绍一个 Claude  Code 里很多人不知道的隐藏指令——`/insights`。

这不是一个帮你写代码的命令，而是帮你**看清楚自己怎么用 AI 写代码**的命令。

---

## 什么是 `/insights`？

一句话解释：`/insights` 会分析你过去所有 Claude  Code 的使用记录，生成一份 HTML 报告，告诉你——

**你到底会不会用 Claude  Code，问题出在哪，怎么改。**

它不是功能介绍，不是文档教程，而是针对你这个人、你的真实使用习惯，给出的个性化反馈。

---

## 怎么用？

极其简单，在终端里输入：

```bash
claude /insights
```

等待分析完成后，它会生成一个 `report.html` 文件，点开就能看。

如果执行失败，先 `Ctrl + C` 关掉 Claude  Code，重新打开再试一次，确保版本是最新的。

---

## 报告里有什么？

报告分成几个部分，每一部分都很实用：

**① 你的互动风格**

你是属于哪种类型？快速迭代、频繁打断，还是一次给完需求、让 Claude 自己跑？报告会帮你归类，并说明这种风格的优劣势。

**② 你做得好的地方**

列出你高效使用 Claude  Code 的操作模式。比如有人善于拆分任务、有人善于给上下文，这些都会被识别出来。

**③ 你的问题所在（最有价值的部分）**

这里会直接告诉你：Claude 在哪些情况下忽略了你的指令、产出了有 bug 的代码、或者走错了方向——而你没有及时发现。

说白了，**就是帮你找出你和 AI 之间的协作断层。**

**④ 推荐的 CLAUDE.md 配置**

针对你的具体问题，给出可以直接复制粘贴的配置行，改完立刻生效。

**⑤ 自定义 Skills 和 Hooks 建议**

根据你的使用习惯，推荐可复用的指令模板和自动化钩子，帮你把好的做法固定下来。

---

## 最扎心的部分

有人用过之后说，看这份报告的感觉"像是在被一位非常了解你的上级做绩效反馈"——有时候夸你，有时候直接指出你的问题。

比如它可能会说：

> 「你在没有得到具体结果之前就放弃了太多对话。」

这句话看起来简单，背后的意思是：你太容易在 AI 给出第一版结果不满意时就放弃，而不是继续迭代、引导它往正确方向走。

这恰恰是大多数人用 AI 效率低的核心原因。

---

## 多久用一次合适？

不建议每天用，因为用得太频繁，发现的都是重复的问题。

**比较好的节点是：**

- 每隔 2-3 周看一次
- 完成一个项目里程碑之后
- 做完一批功能开发、提交完一轮 PR 之后

把它当成**阶段性复盘工具**，而不是日常检查。

---

## 三句话总结

1. **`/insights` 不是功能教程，是针对你个人的使用习惯分析。**
2. **它最大的价值，是帮你发现自己在和 AI 协作时的盲点。**
3. **看完报告，按照建议改一改 CLAUDE.md，你的效率会有明显提升。**

---

## 结语

大多数人用 Claude  Code，停留在「让它帮我写代码」的阶段。

而 `/insights` 让你看到的是：你和 AI 怎么配合、配合哪里出了问题、怎么修。

这才是从「会用」升级到「用好」的关键一步。

试试看，说不定报告里有你没想到的收获。
